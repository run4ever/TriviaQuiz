Recettes Compose Multiplatform — prêtes pour Claude Code

Chaque maquette de TriviaQuiz Concepts.html se traduit en composants Compose simples et communs aux deux plateformes (commonMain → Android & iOS). Voici les blocs réutilisables, du thème jusqu'aux confettis, plus le détail des deux écrans Accueil & Catégories dans leur version actuelle. Tout repose sur les API d'animation standard de Compose — aucune dépendance lourde.

🧭 Noms d'enum réels — ce document utilise les valeurs exactes de data/Question.kt : GEOGRAPHIE, HISTOIRE, SCIENCES_ET_NATURE, ART_ET_CULTURE, SPORTS, DIVERTISSEMENT. Les libellés viennent déjà de l'extension Category.displayName.
💡 Conseil Claude Code : commence par le thème (§1–2), puis traite la section « Les deux écrans » (A–C) pour l'Accueil et les Catégories, et pioche les recettes d'animation au fil des besoins. Demande « adapte ça à mon code existant dans ui/home / ui/category ». Tout est écrit pour commonMain, donc ré-utilisable tel quel pour la future version iOS.

// ui/theme/Rank.kt (nouveau) — libellé de niveau commun Accueil + Catégories
const val SIGNUP_RATING = 750          // rating d'une catégorie à l'inscription

enum class Rank(val min: Int, val label: String) {
    NOVICE(800, "Novice"), APPRENTI(1000, "Apprenti"), CONFIRME(1200, "Confirmé"),
    EXPERT(1400, "Expert"), MAITRE(1600, "Maître");
    companion object { fun of(r: Int) = entries.lastOrNull { r >= it.min } ?: NOVICE }
}

// Seuils des blocs de l'accueil — AJUSTABLES
const val STRENGTH_MIN = 1200          // « Tes points forts » : rating >= ce seuil
const val WEAKNESS_MAX = 1000          // « Tes axes d'amélioration » : rating <= ce seuil

fun Map<Category,Int>.strengths()  = entries.filter { it.value >= STRENGTH_MIN }.sortedByDescending { it.value }.take(3)
fun Map<Category,Int>.weaknesses() = entries.filter { it.value <= WEAKNESS_MAX }.sortedBy { it.value }.take(3)
À l'inscription toutes les catégories valent 750 (sous Novice, et ≤ WEAKNESS_MAX) : elles apparaissent d'abord dans « axes d'amélioration » jusqu'à dépasser 1000. Chaque bloc se masque si sa liste est vide (if (strengths.isNotEmpty())).

Les animations

3Bouton « chunky » qui s'enfoncemicro-interaction

L'effet le plus rentable : une ombre portée colorée + un enfoncement au toucher. collectIsPressedAsState + animateDpAsState sur un offset, sans ripple.

@Composable
fun ChunkyButton(text: String, color: Color, deep: Color, onClick: () -> Unit) {
    val interaction = remember { MutableInteractionSource() }
    val pressed by interaction.collectIsPressedAsState()
    val offset by animateDpAsState(if (pressed) 4.dp else 0.dp, spring(stiffness = Spring.StiffnessMediumLow))

    Box(
        Modifier
            .offset(y = offset)                              // le bouton descend
            .drawBehind { /* dessine 'deep' 6.dp plus bas = la tranche 3D */ }
            .background(color, RoundedCornerShape(18.dp))
            .clickable(interaction, indication = null, onClick = onClick)
            .padding(horizontal = 26.dp, vertical = 16.dp),
        contentAlignment = Alignment.Center,
    ) { Text(text, fontFamily = Baloo, fontWeight = FontWeight.Bold, color = Color.White) }
}

5Apparition en cascade des réponsesstagger

Chaque réponse entre avec un léger décalage. Un AnimatedVisibility par item, déclenché par un index qui monte dans un LaunchedEffect.

@Composable
fun AnswerList(answers: List<String>) {
    var shown by remember { mutableStateOf(0) }
    LaunchedEffect(answers) {                          // relance à chaque question
        shown = 0
        answers.indices.forEach { delay(90); shown++ }
    }
    Column(verticalArrangement = Arrangement.spacedBy(11.dp)) {
        answers.forEachIndexed { i, a ->
            AnimatedVisibility(
                visible = i < shown,
                enter = slideInVertically { 40 } + fadeIn(),
            ) { AnswerCard(a) }
        }
    }
}

6Feedback couleur bonne / mauvaise réponsecœur du jeu

La carte passe en vert ou rouge en douceur via animateColorAsState, plus un pop (scale) sur la bonne réponse et un shake sur l'erreur.

@Composable
fun AnswerCard(label: String, state: AnswerState) {   // IDLE, CORRECT, WRONG
    val target = when (state) {
        AnswerState.CORRECT -> Color(0xFFD5F2DD)
        AnswerState.WRONG   -> Color(0xFFFBE0E0)
        else -> Color.White
    }
    val bg by animateColorAsState(target, tween(280))
    val scale by animateFloatAsState(
        if (state == AnswerState.CORRECT) 1.04f else 1f,
        spring(dampingRatio = Spring.DampingRatioMediumBouncy),
    )
    Row(Modifier.scale(scale).background(bg, RoundedCornerShape(18.dp)) /* … */) { /* badge + texte */ }
}

// SHAKE sur l'erreur : anime un offset X qui oscille
val shake = remember { Animatable(0f) }
LaunchedEffect(wrong) {
    if (wrong) shake.animateTo(0f, keyframes {
        durationMillis = 400; -7f at 80; 6f at 160; -4f at 240; 0f at 400
    })
}

8Anneau de progression / chronotimer

Un arc dessiné au Canvas, dont la fraction est animée. Pour un compte à rebours, anime de 1f vers 0f en tween(durée).

@Composable
fun ProgressRing(fraction: Float, color: Color) {
    val f by animateFloatAsState(fraction, tween(1000, easing = FastOutSlowInEasing))
    Canvas(Modifier.size(64.dp)) {
        val sw = 7.dp.toPx()
        drawArc(color.copy(alpha = .15f), 0f, 360f, false, style = Stroke(sw))
        drawArc(color, -90f, 360f * f, false, style = Stroke(sw, cap = StrokeCap.Round))
    }
}

9Confettis de victoire100% commonMain

Pas besoin de librairie native (qui casserait sur iOS). Une poignée de particules simulées dans un Canvas + withFrameNanos suffit, et tourne partout.

private data class Conf(var x:Float,var y:Float,val vx:Float,var vy:Float,val c:Color,val r:Float)

@Composable
fun Confetti(palette: List<Color>) {
    val parts = remember { List(40) { Conf(0.5f, 0.15f,
        (Random.nextFloat()-.5f)*.9f, -Random.nextFloat()*.4f,
        palette.random(), Random.nextFloat()*6+4) } }
    var t by remember { mutableStateOf(0f) }
    LaunchedEffect(Unit) { while (true) { withFrameNanos { t += 0.016f }
        parts.forEach { it.x += it.vx*.016f; it.vy += .9f*.016f; it.y += it.vy*.016f } } }
    Canvas(Modifier.fillMaxSize()) {
        parts.forEach { drawCircle(it.c, it.r, Offset(it.x*size.width, it.y*size.height)) }
    }
}

Le classement de partie (multijoueur)

10Solo sans fin, multi qui s'arrête

Le mode solo reste volontairement sans fin — aucun changement à faire. C'est la partie multijoueur qui se termine : après un nombre fixé de questions, ou quand l'hôte décide d'arrêter. C'est elle qui porte le classement.

Score de partie — distinct de l'ELO : des points par partie (bonne réponse × rapidité). C'est ce score qui trie le podium.

Entre 2 questions — émets standings (liste triée) ; l'UI affiche le classement provisoire, puis l'hôte appelle nextQuestion().

Fin de partie — sur questionIndex == total ou host.endGame(), émets Finished : même écran, état final + confettis (§9) + gain ELO animé (§7).

// MultiplayerViewModel — pseudo-diff
data class Standing(val player: Player, val points: Int, val correct: Int)

fun onRoundComplete() {
    val standings = players.map { Standing(it, it.points, it.correct) }.sortedByDescending { it.points }
    _state.value = if (questionIndex + 1 >= totalQuestions || host.ended)
        MultiState.Finished(standings, ratingDelta = me.rating - startRating)
    else
        MultiState.Standings(standings, questionIndex + 1, totalQuestions)   // classement provisoire
}

fun nextQuestion() { /* hôte uniquement */ questionIndex++; loadQuestion() }
